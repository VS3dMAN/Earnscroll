chosen the following strategy for v1.0:

PHASE 2 — Account deletion (Tier 1 #2)
--------------------------------------------------------------------------------
Files to create:
  - supabase/functions/delete-account/index.ts
  - app/delete-account.tsx                  (confirmation screen)

Files to edit:
  - app/settings.tsx                        (add Delete Account row above Sign Out)
  - contexts/Auth.tsx                       (add deleteAccount() method)

Edge Function behavior (delete-account):
  - Verifies caller JWT, extracts user_id
  - Runs in this order inside a transaction:
      1. Insert audit row into a new `deletion_log` table (user_id, ts, reason)
      2. DELETE FROM analytics_events WHERE user_id = $1
      3. DELETE FROM analytics_consent WHERE user_id = $1
      4. DELETE FROM user_sessions WHERE user_id = $1
      5. DELETE FROM diagnostic_logs WHERE user_id = $1
      6. (Add any other user-scoped tables — see PHASE 5.1 audit)
      7. supabaseAdmin.auth.admin.deleteUser(user_id)
  - Returns { ok: true, deletedAt } or structured error.
  - Hard-deletes; no soft-delete grace period (simplest for first pass).

In-app flow (app/delete-account.tsx):
  - Reached from Settings → "Delete account"
  - Shows: what will be deleted (Supabase data, analytics, sessions, local
    encrypted prefs); what is retained (anonymized aggregated counts, deletion
    audit row for 30 days per legal requirement); confirmation requires typing
    the word DELETE into a TextInput before the destructive button enables.
  - On success: clears AsyncStorage TimeBank keys, clears EncryptedSharedPrefs
    via a new native bridge method `clearSecurePrefs()`, signs out, navigates
    to /onboarding.

USER ACTION REQUIRED (excluded from execution):
  - Deploy the Edge Function with `supabase functions deploy delete-account`
  - Create the `deletion_log` table via Supabase dashboard (SQL provided in
    plan output for copy-paste)
  - Set service-role key as Edge Function secret

Disclosure screen requirements (matches Google's "prominent disclosure" spec):

      1. What this feature does (1-2 sentences)

      b. From Settings if the user later wants to enable blocking

    need this for an audit response).
