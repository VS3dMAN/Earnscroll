Data Safety Form Guide

A field-by-field reference for filling out EarnScroll's Google Play Console Data Safety section.
Prepared: April 21, 2026

How to use this guide: Open Google Play Console → select EarnScroll → App content → Data safety. Walk through each question in the form and refer to the corresponding section below for the correct answer. Every answer here is based on an audit of the actual EarnScroll codebase.

Section 1: Overview Questions

Q: Does your app collect or share any of the required user data types?

Answer: Yes
EarnScroll collects user data (email, phone for authentication) and stores app activity data locally on the device.

Q: Does your app share any of the required user data types with third parties?

Answer: Yes
Authentication data (email, phone, OAuth tokens) is shared with Supabase (our authentication backend). Font requests are sent to Google Fonts.

Q: Is all of the user data collected by your app encrypted in transit?

Answer: Yes
All network communication uses HTTPS/TLS (Supabase API, Google OAuth, Apple Sign In, Google Fonts CDN).

Q: Do you provide a way for users to request that their data is deleted?

Answer: Yes
Users can: (1) clear all local data via device settings, (2) request account deletion by emailing virajsonib901@gmail.com. Include your account deletion URL/instructions here.

Section 4: AccessibilityService Declaration

CRITICAL: This is a separate form in Play Console under App content → Permissions and APIs → AccessibilityService declaration. You MUST complete this or your app will be rejected.

Q: Does your app use the AccessibilityService API?

Answer: Yes

Q: Describe the core functionality that uses AccessibilityService

Answer: "EarnScroll uses the AccessibilityService API to implement a screen-time management feature. When a user's earned time bank is depleted (reaching zero minutes), the accessibility service detects when the user opens an app from their personal block list and displays a full-screen overlay encouraging the user to earn more time through physical exercise. The service only reads the package name of the foreground application to compare against the user's block list. It does not read screen content, text fields, passwords, notifications, or any other on-screen data. All data remains on the user's device and is never transmitted to external servers."

Q: Is your app an accessibility tool (isAccessibilityTool)?

Answer: No
EarnScroll is not an assistive tool for users with disabilities. The AccessibilityService is used for screen-time management functionality.

Q: Link to your prominent in-app disclosure

Answer: Provide a screenshot or video of your in-app disclosure dialog that:
  1. Appears BEFORE the user is prompted to enable the accessibility service
  2. Clearly explains what data is accessed (foreground app package name only)
  3. Explains the purpose (app blocking when time runs out)
  4. States that data is not transmitted off-device
  5. Requires affirmative user action (e.g., "I Understand" button)
Reminder: The prominent disclosure MUST be shown within the app during normal usage — NOT buried in settings, NOT only in the Privacy Policy or Terms of Service, and NOT combined with other consent dialogs. It must be a standalone, focused disclosure specifically about the accessibility service.

Section 5: Security Practices

Q: Is all of the user data collected by your app encrypted in transit?

Answer: Yes — All network requests use HTTPS (TLS 1.2+)

Q: Do you provide a way for users to request that their data is deleted?

Answer: Yes

• Account data: Users can request account deletion via email (virajsonib901@gmail.com)
• Important: Google now requires a data deletion URL or instructions. Provide a web page or in-app flow for account deletion.

Answer: No (optional field — leave unchecked unless you've completed MASA or similar certification)

Pre-Submission Checklist

  1. Privacy Policy URL — host privacy-policy.html at a public, accessible URL (not a PDF, not geo-fenced). Enter this URL in Play Console under Store listing → Privacy policy.
  2. In-app Privacy Policy link — ensure the Privacy Policy is accessible from within the app (settings screen recommended).
  3. Accessibility Service disclosure — implement a prominent in-app dialog shown before requesting the permission. Take a screenshot for the Play Console declaration.
  4. Account deletion mechanism — provide a clear way for users to request account deletion (email or in-app flow). Enter the deletion URL/instructions in Play Console.
  5. Cross-check consistency — verify every data type declared here matches what's described in the Privacy Policy. No contradictions.
  6. Test all URLs — privacy policy, terms of service, and any links in those documents must be live and accessible.
